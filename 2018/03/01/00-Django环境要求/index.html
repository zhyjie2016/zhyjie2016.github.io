00_Django环境要求

发表于 2018-03-01

安装python

检测python是否可以正常运行，在命令行输入python -V,如果输出python的版本号，说明python已经安装成功

#在CMD中输入下列命令
>python -V
Python 3.5.2

虚拟环境

安装virtualenv

>pip install virtualenv

在指定的目录中运行CMD，创建虚拟环境

>virtualenv [虚拟环境文件夹]

建好虚拟环境后需要激活虚拟环境，运行[虚拟环境文件夹]\Scripts\目录下的activate程序激活

>[虚拟文件夹路径]\Scripts\activate
（[虚拟环境目录]）C:\WINDOWS\system32>

看到命令提示符前多了（[虚拟环境目录]），说明已经激活了虚拟环境

安装django

在虚拟环境下进行操作

>pip install django==1.10.6

查看django版本

(blogproject_env) C:\WINDOWS\system32>python
Python 3.5.2 (v3.5.2:4def2a2901a5, Jun 25 2016, 22:18:55) [MSC v.1900 64 bit (AMD64)] on win32
Type "help", "copyright", "credits" or "license" for more information.
>>> import django
>>> print(django.get_version())
1.10.6

建立django工程

django工程(Project)是我们项目代码的容器。

首先在命令行进入到 C:\Users\yangxg\Workspace 这个目录

(blogproject_env) C:\WINDOWS\system32>cd C:\Users\yangxg\Workspace

注意 windows中不同盘符间跳转时，需要先>盘符：跳到目标盘符后再跳转

运行如下命令创建工程：

(blogproject_env) C:\Users\yangxg\Workspace>django-admin startproject blogproject

Hello Django

在工程目录下运行如下命令,在本地开启一个Web服务器

>python manage.py runserver [8000]

端口号默认8000，启动服务器时可以进行设置

(blogproject_env) C:\Users\yangxg\Workspace>cd C:\Users\yangxg\Workspace\blogproject

(blogproject_env) C:\Users\yangxg\Workspace\blogproject>python manage.py runserver
Performing system checks...

System check identified no issues (0 silenced).

You have 13 unapplied migration(s). Your project may not work properly until you apply the migrations for app(s): admin, auth, contenttypes, sessions.
Run 'python manage.py migrate' to apply them.
December 21, 2016 - 20:23:07
Django version 1.10.4, using settings 'blogproject.settings'
Starting development server at http://127.0.0.1:8000/
Quit the server with CTRL-BREAK.

在浏览器输入127.0.0.1：8000或localhost：8000 可以看到django已经开始工作了

django语言和时区设置

打开setting.py文件，找到如下两行代码：

blogproject/blogproject/settings.py

## 其它配置代码...

LANGUAGE_CODE = 'en-us'
TIME_ZONE = 'UTC'

## 其它配置代码...

改为如下配置

blogproject/blogproject/settings.py

## 其它配置代码...

# 把英文改为中文
LANGUAGE_CODE = 'zh-hans'

# 把国际时区改为中国时区
TIME_ZONE = 'Asia/Shanghai'

## 其它配置代码...

保存更改后关闭setting.py文件.再次启动服务器，可以看到django支持中文了
